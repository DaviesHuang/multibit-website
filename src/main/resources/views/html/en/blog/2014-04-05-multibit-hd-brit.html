Introducing BRIT

What happens when an open source project starts to scale? Costs go up. This article describes how we decided to use Bitcoin to address this problem in what we consider to be the fairest way possible. We call it the Burton-Rowe Income Technique, or BRIT.

Executive summary

With MultiBit HD we're introducing a "client fee" of 1,000 satoshis per spend. The current miner's fee is 10,000 satoshis per 1Kb of transaction storage size. This fee will go towards further development of MultiBit HD and will also be used to support upstream projects that MultiBit HD relies upon (such as Bitcoinj, XChange and so on). By charging a client fee we can avoid intrusive or privacy-breaking alternatives.

Background

MultiBit has been in the Bitcoin space since 2011. In that time it has seen continuous development and improvement so that it is now considered one of the top Bitcoin wallets. Recently we celebrated 1.5million downloads and supporting that size of user base requires either very high reliability or a full time support team. We went for the high reliability approach.

Looking back to October 2013 we made the difficult decision to begin a rewrite of MultiBit Classic and MultiBit HD was born. We chose this path because it was clear that in order for Bitcoin to grow it was imperative that wallet handling had to be made much simpler for non-technical users. This meant ditching random private keys and moving to hierarchical deterministic (HD) wallets based on the BIP32 and BIP39 standards.

We took a lot of the battle-hardened code from MultiBit Classic and improved it. We completely redesigned the user interface so that it would be easy for anyone to use. We laid the groundwork for supporting the Payment Protocol, external hardware wallets (Trezor etc), multi-signature and more. All this development has taken months of full time work, often 7 days a week, and we're only now approaching a first release in May.

Then we faced a hard fact: how do we get paid for all this effort ? We're a wallet not an exchange.

So we made a list laying out our constraints:

  • open source under MIT license
  • no gathering or selling information to third parties
  • no in-app advertising
  • no upfront cost
  • must be pure Bitcoin
  • must survive an upgrade or wallet restore
  • must be decentralised apart from an anonymous initial contact
  • must be extremely low in consideration of people in third world countries
  • must allow income to be allocated to other parties efficiently and anonymously
  • must rely on encrypted messages to prevent snooping, MITM and replay attacks

The result was a client fee of 1,000 satoshis per transaction. This compares favourably with the current miner's fee of 10,000 satoshis per 1Kb (so transactions with a large number of inputs cost more). This would be charged on each spend but because the fee is well below the Bitcoin dust limit of 5432 satoshis, the actual spend won't take place until later. When a sufficient number of client fees have been accumulated the balance is paid off to a Bitcoin address that was obtained during set up.
